## 🏛️ The 4 Layers of Memory

To ensure AI context windows are never overwhelmed, knowledge is structured hierarchically:

### 1. The DNA (Immutable Physics)
These files rarely change and dictate the laws of the universe for the bot.
- `MASTER_CONTEXT.md` — The map of the world (files, db schema, logic).
- `AGENT_RULES.md` — The unbreakable laws every AI must follow.
- `STATUS.md` — The live system health.

### 2. Short-Term Memory / STM (The Active Processor)
What the AI is thinking about *right now*.
- `CURRENT_PLAN.md` — The current sprint and next priorities.

### 3. Medium-Term Memory / MTM (The Recent Past)
What happened over the last few weeks.
- `EXECUTION_LOG.md` — Capped at the 10 most recent sessions.

### 4. Long-Term Memory / LTM (The 1000-Year Archive)
Where old execution logs go to become permanent wisdom.
- `ARCHIVE/DECISIONS_LOG.md` — Distilled, compressed architectural decisions. When `EXECUTION_LOG` gets too long, older entries are compressed and permanently stored here via the **Memory Compression Protocol**.
