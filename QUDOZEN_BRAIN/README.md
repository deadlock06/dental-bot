## 📁 What's In This Folder

| File | Purpose | Update Frequency |
|---|---|---|
| `MASTER_CONTEXT.md` | Complete project map — every file, every function, every connection | When files change |
| `EXECUTION_LOG.md` | Running log of every task executed, what changed, results | After every session |
| `CURRENT_PLAN.md` | The active plan right now — what we're building | When plan changes |
| `IDEAS_BACKLOG.md` | New ideas captured as they come — not yet started | As ideas arrive |
| `AGENT_RULES.md` | Rules every AI agent MUST follow — sacred constraints | Rarely changes |
| `STATUS.md` | System health snapshot — live environment state | After every deploy |

---

## 🤖 If You Are An AI Agent — Read This First

1. **Read `AGENT_RULES.md` before doing ANYTHING**
2. **Read `CURRENT_PLAN.md` to know what we're working on**
3. **Read `MASTER_CONTEXT.md` to understand every file and connection**
4. **Check `EXECUTION_LOG.md` to see what was done last session**
5. **After your work, UPDATE `EXECUTION_LOG.md` with what you did**
6. **If a new idea comes up during work, ADD it to `IDEAS_BACKLOG.md`**
7. **Never work blindly. Never assume. Always check this folder.**

---

## 🔄 Update Protocol (How To Keep This Fresh)

After every work session, an agent or the user must:
1. Add a dated entry to `EXECUTION_LOG.md` with what was done
2. Update `CURRENT_PLAN.md` to reflect next steps
3. Update `STATUS.md` if anything deployed or changed
4. Move completed ideas from `IDEAS_BACKLOG.md` to `EXECUTION_LOG.md`
